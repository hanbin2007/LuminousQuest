好,这块是整个项目真正的技术核心(codex 和我都判断"最难的不是页面"),也是陆老师明确说没思路的部分。我按四个问题讲:标准从哪来、评什么(维度)、怎么调、怎么 eval。

---

## 1. 评分标准从哪来:AI 不定标准,AI 只执行标准

第一原则:**评分标准是化学教研的产物,不是大模型的产物**。来源就四个:2025 修订版课标、教材(选择性必修·化学反应原理)、高考评分惯例(电极反应式的守恒要求等)、陆老师自己的三维模型。

落地流程:**我们起草知识节点图和量表 → 陆老师审定**。她是学科权威,这一步不能省——而且这恰恰是比赛答辩的好答案:评委问"AI 凭什么打分",回答是"标准是教师教研经验编码进系统的,AI 只负责识别学生的话对应标准里的哪一条"。黑箱质疑直接消解。

量表的原子单位是**知识节点**,每个节点长这样(就是 `knowledge-model.json` + `rubrics.json` 的 schema):

```
id: principle.electron-direction
dimension: 原理
statement: 电子经外电路由负极流向正极
evidence_required: 明确说出方向,且区分"外电路"(电子不进溶液)
common_errors:
  - id: e1  电子方向说反
  - id: e2  认为电子经溶液/盐桥传导   ← 高频典型误区
weight: 2
```

首版规模预估:三个维度合计 **15~20 个节点**,每个节点 2~4 个典型误区。不要贪多,节点越多边界越糊。

## 2. 评什么:三级判定,AI 只做中间一级

关键架构决策(比"用什么 prompt"重要得多):**AI 不输出分数**。

1. **确定性层(程序)**:搭建器的结构判分(组件、连接、箭头方向)、判断/选择题——纯规则,零 AI,零调试成本。
2. **抽取层(AI)**:简答和开放分析,LLM 做的是**证据抽取**,输出每个节点的 `hit / partial / miss / 命中误区e2`,并且**必须附学生原文的引用片段**。这个引用可以程序校验(是不是原文子串)——对不上就判抽取无效、自动重试,幻觉在链路上就被拦住了。
3. **判分层(程序)**:extraction × 节点权重 → 分数、雷达图、点亮。每一分都能指回"哪个节点 + 学生原话哪一句"。

这样"AI 评分准不准"就从一个玄学问题变成一个**可度量的分类问题**:抽取的节点判定对不对。调试和 eval 全部围绕它。

另外注意区分两类"维度":雷达图的三个教学维度是**结果性**的;训练环节还有**过程性**指标(用了第几级提示、修正用了几轮)——它们喂给脚手架策略,不进分数,别混在一张表里。

## 3. 怎么调:golden set 驱动的迭代循环

调试的前提是有标注样本(golden set),三个来源按价值排序:

1. **陆老师收真实学生作答**(最值钱):让她开学初拿前测题给学生做一轮,哪怕手写拍照转录。真实的"学生腔"——口语化、半对半错、答非所问——是合成数据模仿不像的。
2. **你自己写**:你刚高考完,就是目标用户画像本人,你写 20 条"故意犯 e2 错误的作答""只答一半的作答",比模型合成的真实得多。
3. **合成补边界**:每个 common_error 至少一条,再补空答、超纲但正确的表述、错别字、把三个维度混在一起答的。

每条样本人工标注期望的 extraction 结果。首版规模:**前测 3 题 × 每题 30~50 条**就够跑起来,不用堆量。

调试循环:跑 eval → 看失败案例 → 判断改哪层。这里有个容易踩的坑:**抽取错不一定是 prompt 的锅,经常是量表本身有歧义**(两个节点边界不清、statement 措辞模型理解不了)。所以量表和 prompt 是一起迭代的,eval 失败案例同时也是给陆老师审量表的素材。选国内哪家 API,也用同一套 eval 跑分说话,不凭感觉。

## 4. Eval 的具体形态

- `eval/cases/*.json`(作答 + 期望 extraction)+ 一个跑批脚本,**走和生产完全同一条链路**(同 prompts、同 provider 抽象、temperature 调低),每个 case 跑 3 次看一致性。
- 核心指标:节点判定准确率(分维度看)、误区识别命中率、**"把全错判成掌握"的比例单独盯**(这是最伤教学信任的错误,门槛建议 ≤2%,其余准确率门槛 ≥90%)、引用幻觉率、JSON 校验失败率、时延与成本。
- **回归纪律**:改 prompt、改量表、换模型,必须重跑全量 eval。所有响应自动录制,录下来的既是免费可重复的回归基线,也是断网演示的回放素材——一套录制机制两用。
- **苏格拉底微循环单独对待**:它是多轮生成式的,不适合精确匹配。MVP 用 checklist 式检查(不泄答案、不跳知识点、追问和学生错误相关)+ 人工抽查——反正只有 3 个案例,人工过得完,别在这上面建复杂的 LLM-judge 体系。
- 手绘彩蛋不判分,不进 eval 门槛,只保证演示案例稳定。
