## M1b.1 修复清单(合并去重)

**阻断/高**
1. 逐槽接地:每个事实槽绑定独立引用;槽值须由确定性别名表/解析器验证"引用文本确实表达该值",验证不过→needs-review;按节点生成闭集事实 schema,禁止重复槽。(codex#1)
2. 语义关键 token 保护:否定词(不/非/未)、正负、氧化/还原、流入/流出、化学式等强制精确一致;非错别字映射内的编辑一律 needs-review 不自动放行;补真实默认阈值(0.12)的边界测试。(codex#2)
3. 辅导泄题加固(裁决:保留生成式,不做全模板化):提示词不再携带完整参考答案(只给误区+情境);tutorable 节点显式配置且要求非空防泄事实集(方程式/builder 节点默认不可辅导,除非配置);泄题检测叠加确定性事实值命中检测(acceptedValues);泄题红队测试:同义改写、代称、拆字、完整方程式。(codex#3+opus#1/#2)
4. 辅导状态服务端化:tutor cycle/turn/terminal 事件持久化进 session,轮数与时限由事件推导,不信任调用方;学生思考期暂停 AI 时限;终态幂等;未给出辅导返回 none/0 消除 schema 冲突。(codex#4+opus#5)
5. 防谄媚硬化:objectiveOutcome=miss 时确定性禁用"肯定正确/无需修改"类结论(禁用词族规则),命中即替换预设;嘴硬测试改为 provider 主动返回谄媚内容,断言被替换且最终 miss。(codex#5)
6. 生产路由接线:新增服务端持有 config/prompt/schema/状态的 /api/assessment/extract 与 /api/tutor/turn;前端不可绕过;六类故障经真实 HTTP 路由注入并验证 session/回放/预设结果。(codex#6)

**中/低**
7. §15/§16 归属编码清晰(preset hint vs socratic correction),用不同配置值做契约测试;补"原始 miss→三轮→修订作答→抽取→判分→session"端到端。(codex#8)
8. eval 候选(裁决:保留完整作答文本):脱敏模式扩充(QQ/微信/地址/学校班级/短学号),落盘完整分类 provenance(config digest、阈值、prompt/schema/model)与稳定 hash;修正现存自相矛盾样本;文档标注"对外分发前须人工审计"。(codex#7/#9+opus#6)
9. 泄题重合率分母:加绝对共享 bigram 下限,防误杀短辅导语。(opus#3)
10. 作答长度上限,防模糊匹配 O(n·L³) 时延。(opus#4)

## 对 codex 两项建议的改判理由

- 全模板化辅导输出:牺牲产品核心卖点(生成式启发提示),且判分确定性已保证泄题不污染分数;采用"不喂答案+确定性事实值拦截+红队测试"的加固生成式方案,残余风险(释义泄题)记录为已知限制,由 M1c 的 checklist 抽查覆盖。
- eval 候选仅存最小片段:与 golden set 需要完整作答原文冲突;本地单机+匿名 ID 前提下保留全文,以扩充脱敏+分发审计门槛替代。
