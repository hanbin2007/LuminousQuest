# 电化学思维模型 AI 诊断训练工具 —— MVP 设计规格

日期:2026-07-14
状态:已与需求方(张涵彬)逐节确认定稿;2026-07-14 完成 gpt-5.6-sol(max)+ fable-5 双路复审并落地修订;**2026-07-15 陆老师认可,进入实现阶段**
背景文档:《人工智能应用能力.docx》(陆老师需求)、江苏省教师基本功比赛规则(AI 应用能力展示环节)

## 0. 一句话定位

给陆老师 2026 年 9 月江苏省教师基本功比赛"人工智能应用能力展示"环节开发的**电化学思维模型 AI 诊断训练工具**:本地运行的网页应用,支撑"2 分钟介绍 + 5 分钟教学演示 + 3 分钟答辩"的比赛流程(总时长 ≤10 分钟,选手自备笔记本演示),同时可供高三学生真实使用。

教学主线(比赛叙事核心):

> 前测收集证据 → AI 定位薄弱维度 → 针对性脚手架训练 → 提示逐步撤除 → 陌生电池迁移验证

## 1. 关键约束与已定决策

| 决策点 | 结论 |
|---|---|
| 前测画图形式 | 双路:拖拽搭建器为主(参与判分),自由手绘为演示彩蛋(仅 AI 点评,不判分) |
| LLM 接入 | 先只接国内 API(DeepSeek / 通义 / 智谱,按手绘识别效果择一),provider 抽象层保证可换 |
| 配置化 | 配置文件(JSON)驱动,开发者修改,MVP 不做教师配置界面 |
| 3D 定位 | 服务于模块二的结果反馈(陆老师定调),但作为评委视觉记忆点**要做精致**——范围限定为一个三轴知识网络,不做三维场景 |
| Agent 架构 | 不上完整 Agent Loop;固定教学状态机 + 受控苏格拉底微循环(≤3 轮) |
| 视觉素材 | 全部拖拽组件与案例素材图由 codex(gpt-5.6-sol)生成**透明底位图(PNG)**,教辅彩色插图风格、本质为抽象简图 |
| 离线预案 | 硬需求:断网可完整跑通演示(LLM 录制回放) |
| 首版范围 | 仅原电池;3 个递进案例(锌铜 → 铝-空气 → 氢氧燃料电池);电解池留二期 |

## 2. 技术架构

**方案:前端 SPA + 本地轻量 Node 服务**(评估过纯前端直连与 Tauri 打包,分别因国内 API 无浏览器 CORS/key 暴露、打包链路过重而排除)。

- 前端:React + TypeScript + Vite;拖拽搭建器 SVG + dnd;手绘 canvas;雷达图 ECharts;3D 用 Three.js(react-three-fiber)。
- Node 服务(Hono 或 Express,极简)只做三件事:
  1. 托管前端静态资源;
  2. 代理 LLM 请求(API key 在 `.env`,不进前端);
  3. 热加载 `config/` JSON 配置 + LLM 请求/响应自动录制与回放。
- 无数据库:学生会话存 localStorage,可导出/导入 JSON。
- 交付形式:双击启动脚本 → 本地服务起在**空闲端口(启动时检测,占用则换,严禁静默抢占)** → 浏览器自动打开。

## 3. 三大模块功能规格

### 模块一:AI 前测诊断

- **拖拽搭建器(主路径)**:组件以**功能化/未命名**形式提供(导体棒、液体容器、连接件、方向箭头、文字标签等),并混入**干扰项**(同种材料的两根电极、非电解质溶液如蔗糖水),避免组件清单本身提示答案——测"选择与建构",不只测"组装"。判分口径与量表 v1 统一:按**功能四要素**(失电子场所/电子导体/离子导体/得电子场所)的齐全性+连接拓扑+方向一致性做确定性判分,盐桥不作为独立必备件;并做**抽象度检查**——电极绑定具体材料(如标注 Zn/Cu)视为"画了一个实例而非通用模型",判 partial 并定向点评(口径见量表裁量 §19b)。
- **3 道针对性题目**(简答/判断,配置文件定义):覆盖原理维度(电极反应、电子/离子流向)与能量维度(能量转化认知)。仅靠画图无法诊断这两个维度,此为对原需求的已确认修正。
- **手绘彩蛋(演示路径)**:自由手绘画布,提交给视觉模型做自然语言点评,展示多模态能力;不参与正式判分。
- **输出**:三维度雷达图 + 每维度证据清单(答对了什么 / 缺了什么 / 命中的典型误区),每一项可追溯到量表条目。

### 模块二:思维模型训练(核心重点)

- 左侧素材区按"抽象 → 现实"递进:陌生电池**装置简图** → **结构剖面图**(实物图 + 真实参数留二期)。
- 3 个递进训练案例全部走配置:锌铜电池(热身校准)→ 铝-空气电池 → 氢氧燃料电池。换题 = 改 JSON,不改代码。
- **冷迁移后测(第 4 案例)**:一个从未用于训练的陌生电池(如甲烷燃料电池),强制三级脚手架、关闭苏格拉底提示,作为"训练前后雷达图对比"的后测依据——教学主线"陌生电池迁移验证"的落点。该案例同时兼作验收标准 2 的实测(只加 JSON 不改代码)。演示模式可直接加载到迁移前状态以适应 5 分钟时限。
- 右侧三维度分析框架,**三级脚手架逐步撤除**:
  1. 框架内引导问题详细;
  2. 仅三个维度空标题,学生自写要点;
  3. 单个空白文本框,独立组织三维度分析。
  脚手架等级由程序按学生画像**确定性**决定(策略参数在配置中),不由 AI 即兴决定。
- **苏格拉底式辅导微循环**(约束硬编码):最多 3 轮;动作仅限追问 / 提示 / 检查修正;不直接公布答案;不跳出当前知识点;LLM 超时或输出异常时回退到配置中的预设提示;最终评分仍由规则与量表完成。
- 即时反馈固定结构:答对了什么 / 错在哪里 / 下一步想什么。

### 模块三:3D 思维模型外显

- 三轴知识网络(装置 / 原理 / 能量),节点与 `knowledge-model.json` 一一对应。
- 学生每答对一个知识点,对应节点实时点亮(发光动效);全部完成呈现完整思维模型。
- 点亮粒度:hit 全亮 / partial 半亮 / miss 暗置(最终口径随量表裁量 §11);**前测结果作为 3D 的初始点亮状态**,训练过程在其上生长。
- 可旋转、缩放;与训练前后雷达图对比联动。
- 打磨标准:评委 30 秒内看懂并留下印象;范围红线:一个三轴网络,不扩展为三维场景。

### 教师视图

- 每个学生的诊断证据链、训练过程记录、脚手架等级变化轨迹——直接回答评委"AI 凭什么打这个分"。
- 会话导出/导入 JSON,会话带匿名学生编号。
- **班级汇总(需求原文"每个学生(或全班)"的全班侧)**:批量导入多份学生会话 JSON → 班级三维度雷达图(均值+分布)、知识节点错误率排行、高频典型误区 Top N。纯前端聚合,不引入账号系统与数据库;这是教师"复习有的放矢"的直接工具,也是答辩"AI 如何改变教师决策"的论据。

## 4. AI 架构

固定教学状态机(不上完整 Agent Loop):

```
学生作答
  → AI 提取知识点(结构化 JSON 输出)
  → 规则与评分量表校验判分
  → 更新学生三维能力画像
  → 程序按确定性策略选择下一题与提示等级
  → AI 生成针对性反馈
  → 最多重试 2 次,进入下一阶段
```

职责分工:

- **AI 负责理解和表达**:识别图像、理解自然语言作答、解释错误、生成启发式提示。
- **程序负责决策**:评分、节点点亮、脚手架等级、是否过关、选下一题。
- **知识量表负责兜底**:电极判断、电子/离子方向、能量转化等关键判定不交给大模型自由裁量。

LLM 接入:国内 API(具体厂商按手绘识别效果与接入体验确定);`LLMProvider` 抽象接口(chat / vision / structured-output 三能力),厂商实现可插拔;所有调用经 Node 代理并自动录制。

## 5. 视觉素材规范

- **所有拖拽搭建器组件、训练案例的装置简图与结构剖面图,均由 codex(gpt-5.6-sol)生成,产物为透明背景位图(PNG)**,透明化处理由 codex 自行负责;不使用外部图库。
- 风格:**教辅彩色插图风**——明快色彩、清晰描边、圆润造型;**本质仍是抽象后的教学简图**(示意性、非写实),教学指向优先于装饰。
- 统一风格规范(色板、描边、造型语言、出图 prompt 模板与参考图)写入 `assets/STYLE.md`,由 Fable 5 制定;位图逐批生成易发生风格漂移,每批素材须对照规范验收(Fable 5 终审素材一致性)。
- 技术要求:透明 PNG,2x 分辨率(适配高分屏);拖拽组件的碰撞/吸附区域在组件配置中单独定义,不依赖图片像素。
- 案例配置通过路径引用素材,换图不改代码。

## 6. 配置体系

全部教学内容落在 `config/` 下的结构化 JSON,Node 端热加载,改完刷新即生效,schema 按"二期可加配置界面"设计:

- `knowledge-model.json` —— 三维度知识节点图:节点、常见错误、诊断依据、3D 坐标。
- `rubrics.json` —— 评分量表:每节点得分规则、证据要求。
- `pretest.json` —— 前测完整定义:搭建器组件集、结构判分规则、3 道题目及各自量表。
- `cases/*.json` —— 训练案例:素材图引用、各脚手架等级引导问题、参考答案要点。
- `prompts/*.md` —— 全部 LLM 提示词模板。
- `scaffold-policy.json` —— 脚手架升降级与选题策略参数。

## 7. 演示与离线预案(硬需求)

- **演示模式开关**:预置与 5 分钟演示脚本对应的学生会话;LLM 响应走录制回放,断网可完整跑通;现场网络可用则实时调用,不可用一键切回放,界面表现无差别。
- 规则引擎、判分、雷达图、3D 本身全离线,不依赖网络。
- 交付物升级为**比赛运行手册**,覆盖全部 10 分钟:2 分钟介绍稿(教学问题+AI 价值陈述)、5 分钟逐步点击脚本(含备用跳转点)、3 分钟答辩高频问答与证据截图包(判分溯源、量表签核、教师决策记录),外加启动/切换演示模式/常见故障处置。
- 交付前在目标笔记本上做**断网全程 10 分钟计时彩排 ≥3 次**(含 2 分钟介绍与 3 分钟模拟答辩),不只是 5 分钟演示段。

## 8. 错误处理

- LLM 调用:超时/异常 → 自动重试 1 次 → 回退录制回放或预设提示,流程永不卡死。
- LLM 结构化输出:JSON schema 校验失败 → 重试;再失败 → 该项降级为"待教师复核",不阻塞学生流程。
- 配置加载:schema 校验,启动时报具体错误(文件、字段、原因)。
- 会话:localStorage 每步自动保存,刷新/崩溃可恢复。

## 9. 测试

- 规则引擎与判分:单元测试(结构判分、量表匹配、脚手架策略)——确定性逻辑必须全覆盖。
- LLM 链路:用录制的真实响应做回放测试,校验结构化提取与降级路径。
- 端到端:断网状态跑通完整演示脚本作为发布前固定验收项。

## 10. 非目标(MVP 明确不做)


电解池、多学生账号系统、后端数据库、教师配置界面、实物电池图片与真实参数、任何形式的完整 Agent Loop。

## 12. 验收标准

1. 断网状态下完整跑通 5 分钟演示脚本(交付前全程 10 分钟计时彩排 ≥3 次)。
2. 换一道题 / 改一条量表只需改 JSON,不碰代码(以第 4 冷迁移案例"只加 JSON 上线"作实测)。
3. 每个分数都能在教师视图中追溯到量表条目与学生作答证据。
4. 苏格拉底微循环在任何异常下都能回退,不卡死流程。
5. 全套位图素材风格统一,符合教辅彩色插图规范(`assets/STYLE.md`)。
6. 班级汇总视图能从 ≥3 份学生会话 JSON 生成班级雷达图与误区排行。
7. 冷迁移案例在无提示条件下可完整作答,并生成训练前后雷达图对比。

## 13. 过程与轻量事项(评审补充)

- **教师决策证据链**(答辩用,支撑"指导开发"叙事):裁量清单的陆老师签核记录、量表变更记录、素材/案例审定记录汇编成一页"教师教学决策日志"——区分"教师的教学设计与裁决"和"AI/开发者的实现"。
- **毕业生可用性测试**:比赛前安排 2~3 名应届毕业生完整走一遍前测→三级训练→冷迁移,记录误解点、操作困难、提示是否泄题,结论回流量表与 eval。
- **演示话术口径**:介绍稿中"实际电池桥接"统一表述为"陌生真实电池类型 + 抽象简图→结构剖面递进",实物图与真实参数明示为二期路线——把裁剪讲成路线图。
