# 3D 性能 Spike 报告(门禁 3 关闭)

日期:2026-07-15 | 执行:Fable 5(亲自)
方案:emissive 材质 + 径向渐变精灵(AdditiveBlending)伪造发光,**不使用 bloom 全屏后处理**;three.js r160。
场景:三轴(装置横/原理纵/能量斜)、24 节点(12 点亮,含呼吸动画)、40 条边、雾效、旋转相机、antialias、dpr=2。

## 结果

- Apple Silicon(M 系列)+ Chrome:活跃渲染 **108–110fps**(ProMotion 屏,贴刷新率),该负载无任何性能风险。
- 测量注意事项(记录给 M4 复测):Chrome 对非前台窗口将 rAF 节流至 ~1Hz,持续基准测量必须保证窗口前台;统计需过滤 >500ms 的节流间隙。
- 视觉结论:精灵光晕方案的发光观感成立(暗夜蓝 #0a1526 上三维度色 glow 层次分明),bloom 不需要——M4 按此方案实装。
- 目标机:陆老师为 M 系列 Mac(同档),风险解除;M4 验收保留"交付机实测 ≥30fps"作为最终确认。

## 决定

1. M4 3D 使用 emissive+精灵光晕,禁用全屏后处理(锁定)。
2. 节点呼吸动画用 sprite 透明度正弦,成本可忽略。
3. 帧率监测组件进 M4(开发用 HUD,生产隐藏),持续测量带节流间隙过滤。
