# M1a 电极反应式 Grammar 冻结

日期：2026-07-15  
版本：`equation-grammar.v1`  
适用范围：高中原电池的半反应式与总反应式确定性判分；不处理有机结构式、配位异构、氧化数自动推断或任意反应预测。

## 1. 输入语法

```ebnf
equation     = side, arrow, side ;
side         = term, { separator, term } ;
term         = [ coefficient ], species, [ state ] ;
species      = formula, [ charge ] | electron ;
formula      = group, { group } ;
group        = element, [ subscript ]
             | "(", formula, ")", [ subscript ]
             | "[", formula, "]", [ subscript ] ;
element      = upper, [ lower ] ;
coefficient  = positive-integer ;
subscript    = positive-integer ;
charge       = "^", [ positive-integer ], ( "+" | "-" )
             | implicit-charge-sign ;
electron     = "e^-" | "e-" | "e⁻" ;
state        = "(s)" | "(l)" | "(g)" | "(aq)" ;
arrow        = "->" | "→" | "=" | "⇌" ;
separator    = "+" ;
```

约束与消歧：

- 接受普通数字、Unicode 上下标数字、`+/-` 与 `⁺/⁻`，进入 parser 前统一为 ASCII。
- 推荐显式电荷写法 `Cu^2+`、`SO4^2-`。兼容高中常见的 `Cu2+`、`Zn2+`：仅当物种是单原子离子时，末尾数字解释为电荷数；`NH4+` 的 `4` 仍是下标，电荷为 `+1`。
- 正离子后的加号与物种分隔符必须可消歧：写作 `Cu^2+ + 2e^-`，或紧凑写作 `Cu2++2e-`。不接受无法唯一解释的拼接。
- 物态可写但不要求；大小写区分元素，`AL` 先由上游笔误归一化策略处理，parser 不把它自动改成 `Al`。
- 系数和下标必须为正整数；不支持小数系数、分数系数、水合点、自由文本条件或箭头上下标注。

## 2. Canonical form

成功解析后按以下顺序生成 canonical form：

1. 展开圆括号和方括号，得到每个物种的元素计数与净电荷；电子表示为独立物种，电荷 `-1`。
2. 丢弃物态标注；`=`、`→`、`⇌` 全部归一为有向 `->`，但不反转左右两侧。
3. 合并同侧相同“元素组成 + 净电荷”的物种；消去两侧完全相同的物种及公共系数。
4. 所有剩余整数系数除以最大公约数，化到最简整数比。
5. 每侧按稳定物种键排序。物种键使用 Hill 顺序的元素计数加净电荷；因此书写次序和整式倍乘不影响等价性。

例：`2Cu^2+ + 4e^- = 2Cu(s)` 与 `4e^- + 2Cu²⁺ → 2Cu` 具有相同 canonical form；反写为 `Cu -> Cu^2+ + 2e^-` 不等价。

canonicalization 只证明符号组成等价，不证明该反应在给定条件下真实发生；介质和案例事实由后续两层校验负责。

## 3. 三守恒校验

单个半反应依次输出三份可溯源结果：

1. **元素守恒**：除电子外，逐元素比较左右总原子数，并返回非零差量。
2. **电荷守恒**：按物种系数乘净电荷，比较左右总电荷，并返回两侧总值。
3. **电子守恒/方向**：半反应中电子必须只出现在一侧，系数为正，并与案例配置的 `expectedElectronSide` 一致；负极氧化默认电子在产物侧，正极还原默认在反应物侧。总反应要求两侧均无电子。成对半反应还要比较电子数；若书写倍数不同，返回最小公倍数和所需倍乘，但“已配平的两式”判定不通过。

元素和电荷守恒不能替代电子方向检查。例如反向书写的守恒式仍会因电极方向不符而被拒绝。

## 4. 介质匹配

介质先做通用禁用检查，再与 `cases/*.json` 中该电极的 `accepted` 等价集比对：

| 介质 | 通用最终式约束 |
|---|---|
| `acidic` | 不允许最终简式残留 `OH^-`；允许 `H^+`、`H2O` |
| `alkaline` | 不允许最终简式残留 `H^+` 或 `H3O^+`；允许 `OH^-`、`H2O` |
| `neutral` | 不设置 H⁺/OH⁻ 的一刀切禁令，由案例等价集决定，因为局部电极反应可能生成其中之一 |
| `molten` | 不允许 `H2O`、`H^+`、`H3O^+`、`OH^-` 作为配平物种 |

碱性铝-空气负极的冻结等价集同时接受：

```text
Al + 4OH^- -> AlO2^- + 2H2O + 3e^-
Al + 4OH^- -> [Al(OH)4]^- + 3e^-
```

二者都必须先通过三守恒；是否按具体教材/电解质条件只保留其中一种，列入 48 小时教师裁决，不阻塞当前实现。

## 5. 判分与 partial 规则

| 情形 | 默认结果 | 证据/回指 |
|---|---|---|
| 解析成功、三守恒通过、介质匹配、canonical form 命中案例等价集 | `hit` | P3 + P6；成对合并正确时再给 P7 |
| 三守恒通过，但使用另一介质的完整守恒式 | `partial` | 定向回指 P3，原因 `medium-mismatch` |
| 物种集合与目标接近，仅有一个系数/单项守恒错误 | `partial` | 回指 P6，并返回元素或电荷差量 |
| 两个及以上守恒项错误、电子在错误侧、产物类别错误或守恒但不属于允许等价式 | `miss` | 回指 P3/P6 的具体失败项 |
| tokenizer/parser 失败 | `miss` | `parse-error`，保留位置和期望 token；不猜测学生原意 |
| 空白、只有“不会”等非方程内容 | `unanswered` | 按裁量 §6 提示重答，不进诊断分母 |
| 只缺物态、使用 `=`、物种顺序不同、整式倍乘 | 不扣分 | 裁量 §9 与 canonicalization |

所有输出都包含原始输入、grammar 版本、解析后的两侧、canonical form（若可生成）、三守恒明细、介质结果、命中的案例等价式和最终 rubric 规则 id。
