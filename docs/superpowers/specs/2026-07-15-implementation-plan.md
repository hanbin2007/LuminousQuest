## 全局架构裁决(评审矛盾项的定论)

- **交付形态**:可执行体 + **外置内容目录**(`config/`、`prompts/`、`assets/`、`recordings/`、`.env`)——打包只封代码与静态资源,外置目录可编辑、热加载,AC2 因此成立。运行时(bun 单文件 vs Node SEA/便携运行时)由 M0 打包 spike 在两平台实测后锁定,全项目统一。
- **会话数据模型进 M0**:版本化 session/event schema——原始作答→量表版本→抽取结果→规则判定→跟随性标记→分数,全链路持久化(AC3 的实现基础);`unassessed`(未测到)与 `miss` 严格分离,雷达分母只含已测节点;匿名学生编号;localStorage 每步自动保存+崩溃恢复;导出/导入 round-trip 测试。
- **缓存/回放键**:provider/model + prompt/schema/config 版本 + 归一化输入 + 图片哈希;启动时校验演示脚本全部 step ID 与资源完整性;cache miss 有兜底路径。
- **安全基线(M0 起)**:服务只绑 127.0.0.1;key 不出 server;录制文件脱敏;学生数据仅本机、匿名编号。
- **手绘彩蛋与判分画像代码级隔离**,不写入正式分数。

## M0 工程骨架 + 三个门禁(7/15–7/18,codex)

- Vite+React+TS 前端、`server/`(Hono)、vitest、pnpm;`/api/llm`(代理+录制回放)、`/api/config`(热加载)、静态托管;空闲端口检测(严禁静默抢占);双击启动脚本(Win/mac)。
- `LLMProvider` 抽象(chat/vision/structured)+ DeepSeek/通义/智谱 adapter + mock provider(无 key 可开发)。
- 全部 config zod schema + 启动校验(最小合法配置可启动;缺文件/空文件/字段错报具体错误)。
- **门禁 1 打包 spike**:两平台"双击跑通 + 改外置 JSON 热生效"实测,锁定运行时。
- **门禁 2 provider 预检**(依赖 API key 到位):能力矩阵(结构化输出稳定性、vision、P95 延迟、单次成本),定主备两家。key 未到位不阻塞其余 M0 项。
- **门禁 3 3D 性能 spike**(Fable 5):核显/目标 Windows 机上 emissive+精灵光晕方案帧率实测(目标 ≥30fps)。
- 验收:两平台冷启动(选端口/开浏览器/错误提示/退出清理)、坏配置报错、打包+外置配置实测通过、录制回放单测通过。

## M1a 确定性核心 + 纵向切片(7/18–7/25,codex)

- **裁量决策表冻结**:23 条逐项记默认值、配置字段、教师裁决截止日;golden set 标注前冻结基线;教材核验疑点 48h 未决走默认值不阻塞(SLA)。
- 教材交叉核验(codex→人工裁决);量表 v1 转录 `knowledge-model.json` + `rubrics.json`(15 节点、3D 坐标继承模型图、D5 跨轴边、依赖与跟随锚点)+ **节点覆盖矩阵**(每节点→题目/案例/证据路径/3D 节点,保证每个节点至少一条可观察证据路径)。
- `pretest.json`(四要素判分+抽象度+干扰项)、`cases/` 前 3 案例、`scaffold-policy.json`、`prompts/`。
- 搭建器拓扑判分引擎;化学式判分器(**先冻结输入 grammar + canonical form + 介质/partial 规则,再建测试语料**:合法等价式/近似错误式/解析失败);画像聚合(可溯源评分记录:分数→rubric 条目→证据原文)。
- **纵向切片验收(7/25)**:锌铜案例"作答→判分→证据→雷达→保存→恢复→离线回放"端到端跑通——集成风险前置暴露。
- 判分逻辑验收口径:规则决策表 100% 覆盖 + 分支覆盖 ≥90% + 拓扑/守恒性质测试。

## M1b 抽取与辅导(7/25–8/1,codex)

- 闭集抽取(node/error id)+ 引用原文校验(归一化模糊匹配、错别字容忍)+ schema 重试 + "待教师复核"降级。
- 苏格拉底微循环:固化超时时限、重试 1 次、动作白名单、泄题检测、回退预设提示、强制推进时限;**故障注入测试清单**:超时/HTTP 错误/非法 JSON/越界动作/泄题命中/回放缺失——逐项测过才算 AC4 有据。

## M1c eval(7/28–8/3,codex,与 M1b 部分并行)

- golden set 规范:总量 ≥150 条、每节点 ≥5 条、每误区 ≥3 条;**独立保留集**(不用于 prompt 迭代);标注=张涵彬,Fable 5 抽查 20% 复核;指标=逐节点宏平均命中率 ≥90%、"全错判掌握"≤2%(混淆矩阵报告分母)、一致性(每 case×3);蜕变测试(改写/换名不变分、加废话不降级)。
- 真实手绘与学生腔样本:7 月底毕业生走查时采集(手绘不判分,视觉选型不阻塞主线)。
