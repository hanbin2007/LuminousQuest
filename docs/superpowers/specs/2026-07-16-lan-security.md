# 局域网模式与安全边界

日期:2026-07-16

## 启动

默认启动仍只监听 `127.0.0.1`，同机以外的设备不能连接:

```bash
./LuminousQuest
```

只有显式传入 `--lan` 才监听 `0.0.0.0`:

```bash
./LuminousQuest --lan
# 源码运行:
pnpm serve -- --lan
```

启动日志会打印本次随机生成的访问令牌和检测到的私网 IPv4 URL，例如:

```text
[startup] LAN access token: <本次随机令牌>
[startup] LAN URL: http://192.168.1.8:4173/?access_token=<本次随机令牌>
```

浏览器首次使用该 URL 时，服务端校验令牌，写入 `HttpOnly; SameSite=Strict` 会话 cookie，并立即跳转到不含令牌的 URL。UI、静态资源和所有 API 均受同一入口令牌保护。令牌不落盘，每次重启都会轮换。

## 安全边界

- 只在可信的家庭/学校私有局域网使用。不要在公共 Wi-Fi 上开启。
- 该模式使用明文 HTTP，不提供 TLS，也不抵御同一网络中的被动监听。访问令牌不是互联网暴露的替代方案。
- 禁止把端口映射到公网、配置路由器端口转发、接入公网反向代理或把 URL/令牌发到公开频道。
- 持有令牌的人可操作课程、触发本机配置的模型调用，并读取当前匿名会话证据。演示结束后立即退出程序，使令牌失效。
- `.env`、模型密钥和外置内容目录仍只保存在主机；浏览器端拿不到 provider key。LAN 访问令牌与内部 API token 是两个独立随机值。
- 若启动日志没有列出私网 URL，先检查主机与展示设备是否在同一网段；不要改成公网地址绕过检测。

## 现场检查

1. 不带令牌访问 `/api/runtime` 应返回 `401`。
2. 使用日志 URL 首次进入后，地址栏中不再保留 `access_token`。
3. 另一浏览器未持有 cookie 时仍应返回 `401`。
4. 关闭应用并重启后，旧 URL 与旧 cookie 均不能继续进入。
