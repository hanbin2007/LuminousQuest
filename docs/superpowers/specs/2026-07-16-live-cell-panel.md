# 训练分屏 · 实时 3D 统一认知模型面板(规格补充)

日期:2026-07-16 | 作者与实现:Fable 5(3D/视觉属亲自负责域) | 依据:陆老师手稿《电化学统一认知模型》(/Users/zhb/Downloads/E8C3B5A4226B219CE23A9A868F019F70.jpg)

## 需求

学生在 AI 引导(模块二训练)过程中,分屏实时呈现一个**抽象 3D 原电池模型**,叠加**三维度分析**(原理维度↑ / 装置维度→ / 能量维度↙),与手稿同构。作答判分、蓝笔追问实时驱动模型点亮与动效。

## 核心映射(全部配置驱动,不新增知识硬编码)

手稿与 `config/knowledge-model.json` 天然同构:

| 手稿元素 | 场景元素 | 驱动数据 |
|---|---|---|
| 左电极条 | 左电极板(负极) | D1 灯态;材质色/名称由本案例 `polarity.assessed`(outcome=hit)事件 correctValue 的 `negative=<token>` 查表,判定前显示「?」(防泄题) |
| 顶部导线 + e⁻ 箭头 | 导线弧 + 电子粒子流 | D2 灯态点亮导线;P4 点亮时电子粒子沿弧流动 |
| 隔膜(虚线)+ 阴/阳离子箭头 | 电解液体 + 虚线隔膜 + 离子漂移箭头 | D3 灯态点亮液体;P4 点亮时离子箭头漂移(阳→正极,阴→负极) |
| 右电极条 | 右电极板(正极) | D4 灯态;同上 `positive=<token>`,判定前「?」 |
| 「得失电子场所双层」 | 电极双层描边环 | D5 灯态 |
| 电流标注 | 导线上电流方向箭头(琥珀,与 e⁻ 反向) | P5 点亮时出现 |
| 原理维度轴(左,纵向) | 原理轨道 + P1–P7 发光标记 | 节点 `position.y/x` 直接取自配置 |
| 能量维度轴(左下对角) | 能量轨道 + E1–E3 标记;释能/储能副标 | E 节点灯态;任一点亮则「释能:化学能→电能」增亮 |
| 装置维度轴(底部) | 底部轨道 + 四类目刻度(失电子场所/电子导体/离子导体/得电子场所) | 对应 D1–D4 灯态增亮 |
| 放电/充电双标注 | 电极角色双行字幕:放电行亮、充电行暗置 | 静态教学标注(v1 仅放电态;充放切换留作后续) |
| +/− 极性 | 电极上方极性徽章 | 本案例 `polarity.assessed` 最新 outcome=hit 点亮 |

灯态语义与模块三完全同源(full-lit/half-lit/dark/unassessed/needs-review),色板同 `.stage-dark` tokens(校青=原理、校蓝=装置、琥珀=能量)。

## 实时性

- 状态推导为纯函数 `buildLiveCellState(session, config, trainingCase)`:取**本案例**全部 `assessment.completed` 事件按节点取最新,映射灯态;点亮次序按事件 sequence(与模块三 ignition 同语言)。
- 提交作答 → session 合并 → 灯态签名变化 → 触发一次逐节点点亮序列(stagger 同模块三 0.08s)。
- 蓝笔追问 / 点击证据批注卡 / 点击面板节点芯片 → `focusNodeId` → 对应场景元素脉冲光晕 + aria-live 播报节点陈述。

## 布局

- `training-page--split` 两栏网格:主列(素材+作答+反馈)| 右侧**常驻**3D 面板,**严格占页面 50%**、粘性满高(分屏页突破常规 1200px 栏宽;用户 2026-07-17 指定比例)。
- <1200px 单列:面板置顶(位于页面标题之前;比赛演示为桌面场景,窄屏为非关键路径,终审决议 #15)。
- 面板含:标题(电化学统一认知模型 · n/N 已点亮)、Canvas、聚焦播报(aria-live)、D/P/E 节点芯片(键盘可选,双向同步选中)、灯态图例。

## 教学信号边界

- **冷迁移后测(transfer 案例)冻结面板**:不渲染任何灯态/点亮/统计,以静态说明占位——后测不得提供即时对错信号(终审决议 #5)。
- 知识模型节点陈述属《统一认知模型》通用教学框架(模块三与前测诊断页已全量公开),非案例答案;可判分秘密(极性绑定/acceptedValues/误区映射)始终门控(终审决议 #2 驳回泄题指控的依据)。

## 约束(延续 M4 决议)

- Canvas `aria-hidden`,可达性由芯片清单与 aria-live 承担;无 WebGL 时渲染文字灯态清单。
- 全部 texture/geometry memoize + dispose;useFrame 零分配;reduced-motion 停周期动画(粒子/漂移/脉冲取稳态)。
- 无后处理 bloom(沿用 emissive+additive sprite 方案)。
- 演示模式零特判:面板纯由 session 驱动,demo 回放自动生效。
- 冻结物不动:本功能为 release/m4-rc1 之后的主线开发,不改 M4 runbook 文档。

## 显示查表(唯一的新增表述层数据)

锚点 token → 电极显示名/材质色(`Zn→锌/锌灰`,`Cu→铜/铜橙`,`Al→铝`,`porous-carbon→多孔碳/石墨深灰`,`hydrogen-Pt→H₂·Pt`,`oxygen-Pt→O₂·Pt`,`methane-side→CH₄ 侧`,`oxygen-side→O₂ 侧`);未收录 token 回退原文与惰性灰。属纯表现层;将来可被案例配置的可选 `display` 字段覆盖(后续里程碑)。

## 验收

1. 训练页分屏呈现 3D 模型,三轴与手稿同构,校色三维度着色。
2. 提交锌铜案例作答后,对应节点实时点亮(含逐个点亮动画);P4 点亮后电子粒子沿导线流动、离子箭头漂移方向正确(阳→正,阴→负)。
3. 蓝笔追问某节点时,场景对应元素出现聚焦脉冲。
4. 切换案例后电极名称/材质随锚点变化(四案例全覆盖)。
5. 无 WebGL(jsdom)回退文字清单;reduced-motion 无周期动画;全量测试绿。
